Selection Sort Deck

這版是 selection sort。它的節奏和 bubble / insertion 不同： 每一回合先固定一個起點，然後在右側未排序區中一路掃描，找出 最小值的位置， 最後再和目前回合的起點交換。重點不是一直相鄰交換，而是「先找最小，再一次換位」。

複雜度摘要

情況 時間 說明
Best Case O(n²) 即使幾乎有序，也還是要掃描剩餘元素找最小值。
Average Case O(n²) 每回合都要做線性搜尋。
Worst Case O(n²) 比較次數與一般情況差不多，主要差在交換結果。
Space O(1) 原地排序。
Stability 通常不穩定 因為直接交換可能改變相同元素的相對順序。
這份程式把事件拆成 round-start / scan / new-min / swap / round-end / done， 讓 selection sort「搜尋最小值」的特徵更明顯，和前兩個排序的動畫邏輯有清楚區隔。

Merge Sort Flow

這版是合併排序。它和前面三種不一樣，核心不是在原陣列上一路交換， 而是先把資料拆分（divide）成更小區段，直到每段只剩一個元素， 再把左右兩段依序合併（merge）回去。真正的重點在於「分割結構」與「合併過程」。

複雜度摘要

情況 時間 說明
Best Case O(n log n) 拆分層數固定，合併時仍需線性處理元素。
Average Case O(n log n) 分治法的典型效率。
Worst Case O(n log n) 比起 bubble / insertion / selection，規模變大時通常更穩定。
Space O(n) 需要額外暫存陣列來做 merge。
Stability 穩定 相等元素可保留相對順序。
這份程式故意把 merge sort 的流程拆成 split / base / merge-start / compare / write / merge-end / done， 讓你看得出來這不是交換型排序，而是典型的 divide and conquer。
